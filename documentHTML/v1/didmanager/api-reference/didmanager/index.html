# DIDmanager

DID manager SDK for manage DIDs

# Main function

# createDID 

dids.createDID(claimKey, data, privateKeySigner, privateKey, nonce);

Create DID contract. In this version, this function only accept claimKey.PATIENT and claimKey.PROVIDER as claimKey, if you want to use new claimKey, please use addClaim function with a custom claimKey

# Parameters

  1. claimKey - PreDefinedClaimKeys : DID type, default is PreDefinedClaimKeys.PATIENT (check DID type - PreDefinedClaimKeys)
  2. data - string : When create DID, we suggest using PharumoID as data for this field
  3. privateKeySigner - string : Private key of CLAIM_SIGNER of claimHolder, this is fixed in backend server side
  4. privateKey - string : Private key of DID contract creator - owner of DID contract which will be created
  5. nonce - number (optional): The nonce of account, use this if you want to use custom nonce

# Returns

  1. receipt - object : A transaction receipt object, or null if no receipt was found. (check receipt)
  2. eventLogs - object : Decoded Event logs of transaction
  3. claimAdded - string : Added claim of DID contract
    • claimId - string : ID of added claim
    • claimKey - string : Key of added claim
    • claimData - string : Data (value) of added claim in hex string, convert it back to string for original data

# Example 

dids.createDID(
    PreDefinedClaimKeys.PATIENT,
    "PID0000001",
    "187bcbef9261e6c7eaefd8368e2b930a8bd7335cf541d8a05e9337beaf4c5f89", // This is an example of CLAIM_SIGNER of ClaimHolder, we will use env.CLAIM_SIGNER_PRIVATE_KEY in other function
    "147ba01ee975ff771486759b75206a128babdef687aed85c15c13dbb28f038cf"
  ).then(console.log);

# addKey 

dids.addKey(purpose, keyType, privateKeyAdd, delegateKey, targeDID, nonce);

Add key to Claim holder (only access for Management side).

"Warning"

In the first time you add key to a DID, that DID has no key execpt MANAGEMENT, you must use that MANAGEMENT key to do this function. After that, you can use MANAGEMENT key or ACTION key to add more key to DID.

# Parameters

  1. purpose - KeyPurposes : The key purpose (check Key purpose)
  2. keyType - Schemes : The key type (check Schemes)
  3. privateKeyAdd - string : The private key of account to add as key
  4. delegateKey - string : Private key of DELEGATE_KEY or MANAGEMENT key in first time
  5. targeDID - string : Target DID contract address
  6. nonce - number (optional): The nonce of account, use this if you want to use custom nonce

# Returns

  1. receipt - object : A transaction receipt object, or null if no receipt was found. (check receipt)
  2. eventLogs - object : Event logs of transaction
  3. keyAdded - string : Added key of DID contract

# Example 

dids.addKey(
    KeyPurposes.DELEGATE_KEY,
    Schemes.ECDSA,
    "dbed7d737aac4bc51f876336183cb73d295c7b9528eb734a69ebf9bbca613cc8", //This is just a sample key, do not use in production

# addClaim 

dids.addClaim(
  delegateKey,
  privateKeySigner,
  targetDID,
  claimKey,
  scheme,
  issuer,
  data,
  [uri],
  nonce
);

Add claim to DID contract

# Parameters

  1. delegateKey - string : Private key of DELEGATE_KEY or MANAGEMENT key
  2. privateKeySigner - string : Private key of account that has permission to interact with DID
  3. targetDID - string : DID contract address
  4. claimKey - string : The key of claim, this can be any string
  5. scheme - Schemes : The scheme of claim (check Schemes)
  6. issuer - string : The issuer address of claim
  7. data - string : The raw data of claim
  8. uri - string (optional): The uri of claim
  9. nonce - number (optional): The nonce of account

# Returns

  1. receipt - object : A transaction receipt object, or null if no receipt was found. (check receipt)
  2. eventLogs - object : Decoded Event logs of transaction
  3. claimAdded - string : Added claim of DID contract
    • claimId - string : ID of added claim
    • claimKey - string : Key of added claim
    • claimData - string : Data (value) of added claim in hex string, convert it back to string for original data

# Example 

dids.addClaim(

  env.CLAIM_SIGNER_PRIVATE_KEY,

  Schemes.ECDSA,
  CONFIG.ClaimHolder.address,

# getClaimById 

dids.getClaimById(accountDID, claimId);

Get claim detail of accountDID by claim id, this will be use to get specific claim if there are 2 claims with same key but from different issuer

# Parameters

  1. accountDID - string : DID contract address
  2. claimId - string : The claim id

# Returns

  1. claimKeys - string : The claim key
  2. scheme - string : The claim scheme, this return a string but it's actually number, please parse it to number
  3. issuer - string : The claim issuer
  4. signature - string : A hex string of the signature
  5. data - string : A hex string of the data
  6. uri - string : URI of the claim

# Example 

dids.getClaimById(
      "0xeAB6D21DC3e1eA9441D20EEDfD9133cf37732724",
      "0xa42a10be08f648452f1d1e956b9c169331da201bd0b331860314c904ca2cce89"

# verifyClaim 

dids.verifyClaim(accountDID, claimKey);

Check if accountDID has a valid claim key issued by our system

# Parameters

  1. accountDID - string : DID contract address
  2. claimKey - claimKey : The claim key

# Returns

boolean : True if accountDID has a valid claim key, false otherwise

# Example 

dids.verifyClaim("0xeAB6D21DC3e1eA9441D20EEDfD9133cf37732724", "Name");
> true
